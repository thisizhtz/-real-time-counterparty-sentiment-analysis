Overview

从文本流到风控动作的一体化页面

页面把项目能力组织成风控人员更易理解的形态，同时保留底层 Python 包、CLI 与 JSONL 事件模型，便于后续接入真实流式数据。

01

可解释评分

基于扩展金融 NLP 词典输出 positive、neutral、negative 标签，并展示命中词、解释摘要和置信度，便于审计与模型风险管理。

02

风险早预警

将 default、missed payment、systemic financial risks、private credit、sanction、fraud 等文本信号映射为标准化风险标签。

03

滚动快照

按交易对手方维护窗口内平均分、最新标签、风险旗标和多维评分，形成实时监控视图。

Workflow

推荐的实时处理链路

  1. 1

    采集

    从新闻、公告、邮件、工单或聊天系统收集文本，统一转成 JSONL 或消息队列事件。

  2. 2

    标准化

    使用 TextEvent 规范 counterparty、source、timestamp、event_id 与 text 字段。

  3. 3

    评分与解释

    执行扩展词典评分、否定词修正、强度词加权、多维风险映射，并保留命中词作为解释证据。

  4. 4

    监控与处置

    将滚动快照推送到仪表盘、预警队列或风控工作流，优先处理负面趋势和高风险旗标。

Deployment

从演示页面到生产系统

当前页面适合演示与需求对齐；生产部署时可以复用 Python 包中的分析器和流式状态管理，并把网页输出替换为消息总线、数据库、BI 看板或告警系统。

下一步建议

  • 将词典外置为经过审批的 JSON/YAML 配置。
  • 接入 Kafka、SQS、Pub/Sub 或内部事件总线。
  • 为不同来源设置阈值、权重与人工复核策略。
  • 增加历史回测、漂移监控、多维阈值校准和审计日志。
